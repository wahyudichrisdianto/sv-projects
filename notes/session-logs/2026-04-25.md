## Completed Work

### 1. Fixed Miniflare SQLite Lock Crash
**Problem:** `SQLITE_BUSY_RECOVERY` error caused dev server to crash on hot-reload.

**Root Cause:** `persist: false` in `svelte.config.js` made Miniflare use an in-memory D1 database, but the emulated D1 SQLite file in `.wrangler/state/v3/` was being locked by stale processes.

**Solution:**
- Removed `persist: false` from `svelte.config.js`
- Used `wrangler d1 execute sv_project --local --file=...` to apply Drizzle migrations to the local D1 SQLite file
- Miniflare now reads from the same persistent SQLite file that wrangler populates
- Added `migrations_dir` config exploration (reverted — wrangler v4 doesn't support it; use `--file` flag instead)

**Files Changed:**
- `svelte.config.js`
- `wrangler.jsonc`
- `src/hooks.server.ts` (reverted setup script)
- Deleted `src/lib/server/db/setup.ts`

### 2. Added Social Links to Testimonials
**Feature:** Allow testimonials to include up to 4 social media links (GitHub, LinkedIn, X, etc.) with auto-detected platform icons.

**Implementation:**
- **Schema:** Added `socialLinks: text('social_links')` to `testimonial` table
- **Migration:** Generated `drizzle/0001_lumpy_lady_mastermind.sql` and applied to local D1
- **API (`+server.ts`):**
  - `GET`: Parses `social_links` JSON string into array
  - `POST`: Accepts `socialLinks: string[]`, validates max 4 URLs, filters empty/non-http, stores as JSON
- **Shared Utility (`src/lib/utils/social.ts`):**
  - `detectPlatform(url)` — detects platform from URL domain
  - `getPlatformIcon(url)` — returns inline SVG for detected platform
  - Supports: GitHub, LinkedIn, X, Facebook, Instagram, YouTube, Dribbble, Behance, fallback link icon
- **Form (`testimonials/new/+page.svelte`):**
  - 4 clean URL input fields with auto-detecting icons on the left
  - Icon changes dynamically as user types (e.g. `github.com` → GitHub icon)
  - Sends `socialLinks: string[]` in POST payload
- **Display (`Testimonials.svelte`):**
  - Renders social icon links below author name/role
  - Icons are clickable, open in new tab
  - Removed public "Leave a testimonial" button (invitation-only access)

### 4. SectionNav Mobile Improvements
**Dynamic Label Wrapping:**
- Added optional `wrap?: string` to `SectionItem` type
- `ResizeObserver` measures dock width and detects if labels would overflow
- If overflow detected AND item has `wrap` → shows short text
- If > 6 items → activates `flex-wrap` with 3-per-row layout
- Desktop unaffected (full labels, vertical dock)

**Home nav config updated:**
```ts
{ id: "profile", label: "Profile", wrap: "Pro" },
{ id: "experience", label: "Experience", wrap: "Exp" },
{ id: "testimonials", label: "Testimonials", wrap: "Tst" },
{ id: "contact", label: "Contact", wrap: "Ctc" },
```

**Scroll Animation Fix:**
- **Problem:** Clicking a nav item caused flicker — button briefly active then lost highlight while scrolling
- **Solution:** Added `pendingId` state
  - Click → sets `pendingId` (text color + indicator, NO background)
  - During scroll → clicked item stays visually highlighted
  - Observer reaches target → sets `activeId` (background appears), clears `pendingId`
  - 1.2s fallback timeout if observer never fires

## Architecture Decisions

- **Social links storage:** Stored as JSON string in SQLite `text` column rather than separate table — simple, fits the use case (max 4 URLs per testimonial).
- **Platform detection:** Pure client-side URL parsing — no external icon library needed, all SVGs are inline.
- **Local D1 workflow:** `drizzle-kit generate` → `wrangler d1 execute --local --file=...` — avoids `persist: false` in-memory issues while keeping data across restarts.
