# sv_project — Project Notes

## Overview
Personal portfolio website for Wahyu Chrisdianto (foohyu_). Built with SvelteKit 5 and deployed to Cloudflare Workers with D1 SQLite database.

---

## Tech Stack

| Layer | Tech |
|-------|------|
| **Framework** | SvelteKit 5 (Svelte 5, runes mode) |
| **Language** | TypeScript 6 |
| **Package Manager** | bun |
| **Styling** | Tailwind CSS v4 |
| **Database** | Cloudflare D1 (SQLite) + Drizzle ORM |
| **Deploy** | Cloudflare Workers (`adapter-cloudflare`) |
| **i18n** | Paraglide (`@inlang/paraglide-js`) |
| **Build** | Vite 8 + Wrangler |

## Design System (Swiss Minimal)

**Palette**: oklch-based, soft contrast. No pure white/black.
- Light mode: bg `oklch(97% ...)`, surface `oklch(99% ...)`, text `oklch(25% ...)`
- Dark mode: bg `oklch(18% ...)`, surface `oklch(22% ...)`, text `oklch(88% ...)`
- All auto-detects `prefers-color-scheme`, with manual `[data-theme]` override

**Typography scale**: xs (0.6875) → sm → base (0.9375) → lg → xl → 2xl (1.75) → 3xl → 4xl → 5xl (4rem)

**Spacing**: `--space-element` (1.5rem), `--space-block` (3rem), `--space-section` (6rem)

**Max content width**: 860px (`.grid-container`)

**Border radius**: minimal (2px / 4px)

**CSS utilities**: `.section`, `.grid-container`, `.divider`, `.label`, `.card-hover`, `.theme-transition`

**Theme store** (`theme.svelte.ts`): 3 modes — `auto` | `light` | `dark`. Stored in localStorage.

## Database (Cloudflare D1 + Drizzle)

### Schema (`src/lib/server/db/schema.ts`)

```ts
// testimonial — user-submitted, no auth required
testimonial: { id, name, role, image (nullable), message, createdAt }

// task — default example table from SvelteKit scaffold
task: { id, title, priority (default 1) }
```

### Key files
- `src/lib/server/db/index.ts` — `getDb(d1: D1Database) → drizzle(d1, { schema })`
- `drizzle.config.ts` — uses `d1-http` driver, reads creds from `.env`
- `wrangler.jsonc` — D1 binding: `DB` → `sv_project` database

### Important: Local Dev D1

**Problem**: `getPlatformProxy` uses **local Miniflare D1** when `remoteBindings: false`. The local DB is at `.wrangler/state/v3/d1/`. `db:push` only pushes to the **remote** D1 via `.env` creds.

**Solution**: When schema changes, manually create tables in the local D1:
```bash
# Find the local DB file
ls .wrangler/state/v3/d1/miniflare-D1DatabaseObject/*.sqlite

# Create tables manually
sqlite3 <local-db-file> "CREATE TABLE IF NOT EXISTS testimonial (...)"
```
Or delete `.wrangler/state/` to reset (creates empty local DB).

### D1 API (direct access)
```bash
# Query D1 directly
curl -X POST "https://api.cloudflare.com/client/v4/accounts/$CLOUDFLARE_ACCOUNT_ID/d1/database/$CLOUDFLARE_DATABASE_ID/query" \
  -H "Authorization: Bearer $CLOUDFLARE_D1_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"sql":"..."}'
```

## Known Gotchas

### 1. Remote proxy session fails
**Error**: `Failed to start the remote proxy session`
**Fix**: `svelte.config.js` uses `platformProxy: { remoteBindings: false }`
**Why**: Wrangler tries to connect to Cloudflare API for remote bindings but fails. Disabling remote bindings uses local Miniflare instead.

### 2. Local D1 empty after schema change
**Problem**: `db:push` only pushes to remote D1. Local D1 (used in dev) stays empty.
**Fix**: Manually create tables in local `.wrangler/state/v3/d1/*.sqlite` or delete `.wrangler/state/` and restart.

### 3. Rerun `wrangler types` after wrangler.jsonc changes
After adding/removing D1 bindings, run `bun run gen` to regenerate `worker-configuration.d.ts`.

### 4. Svelte 5 runes mode
Project uses Svelte 5 runes (`$state`, `$props`, `$derived`). Not signal/class store patterns.
- State: `let x = $state(value)`
- Props: `let { name } = $props()`
- No `$:` or `export let` patterns

### 5. Self-closing HTML tags warning
Svelte 5 warns on self-closing non-void elements like `<span />`. Use `<span></span>` instead. Known warnings exist in `Header.svelte`, `SectionNav.svelte`, `ExperienceTimeline.svelte`.

## Key Component Patterns

- **Layout**: `<Header />` → `<main role="main">{@render children()}</main>` → `<Footer />`
- **Pages**: Use `+page.svelte` for UI, `+page.server.ts` for data loading
- **API routes**: `+server.ts` with `GET`/`POST` exports
- **Components**: Pure Svelte 5, use `$state` for local state, `$props()` for inputs
- **DB access**: Only in server code (`+page.server.ts`, `+server.ts`), uses `platform.env.DB`
- **Graceful fallback**: All DB endpoints return empty/default when `platform?.env?.DB` is missing
